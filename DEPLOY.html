1 Create the project folder structure

Create a folder on your computer called clario. Inside it, create this exact structure:

clario/ ├── api/ │ ├── chat.js ← secure Claude proxy (included in your download) │ └── zodiac.js ← daily card generator (included in your download) ├── public/ │ ├── clario-landing.html │ ├── clario-app.html │ └── clario-dashboard.html └── vercel.json ← routing config (included in your download)
Files included in your download

The deployment package below contains all of these files pre-configured. Download it, unzip it, and you have the complete folder ready to push to GitHub.

2 Push to GitHub

Vercel deploys directly from GitHub. You need to push your project folder to a private GitHub repository.

Option A — GitHub Desktop (easiest, no terminal)

  • Download GitHub Desktop from desktop.github.com
  • Click File → New Repository — name it clario, set to Private
  • Copy your clario folder contents into the repo folder GitHub Desktop created
  • Click Commit to main → then Push origin

Option B — Terminal

cd clario git init git add . git commit -m "Initial Clario deploy" git remote add origin https://github.com/YOUR_USERNAME/clario.git git push -u origin main
Critical — never commit API keys

Your Anthropic API key must NEVER go in any file. It lives only in Vercel's environment variables (Step 4). The files in this package contain no keys — the proxy reads the key from the server environment only.

3 Import to Vercel
  • Go to vercel.com and sign in (use GitHub — it connects automatically)
  • Click Add New → Project
  • Find your clario repository and click Import
  • Framework Preset: leave as Other (not Next.js)
  • Root Directory: leave as / (the repo root)
  • Do NOT click Deploy yet — go to Step 4 first to add environment variables
Vercel auto-detects the api/ folder

Because your project has an api/ folder with .js files, Vercel automatically deploys them as serverless functions. No extra configuration needed.

4 Add environment variables — this is where the API key goes

On the Vercel import screen, before clicking Deploy, scroll down to Environment Variables and add these:

ANTHROPIC_API_KEY Your Anthropic API key. Get it from console.anthropic.com → API Keys → Create key. Starts with sk-ant-api03-...
NODE_ENV Set to: production
Set all 3 environments

When adding ANTHROPIC_API_KEY, make sure all three checkboxes are ticked: Production, Preview, and Development. Otherwise the API won't work in preview deployments.

Now click Deploy. Vercel will build and deploy in about 30 seconds.

After deploy succeeds

Vercel gives you a URL like clario-xyz.vercel.app. Open it — your landing page should load. Test the board session to confirm the API proxy works.

5 Point clarioai.ae to Vercel

Part A — Add domain in Vercel

  • In Vercel dashboard → your project → Settings → Domains
  • Click Add Domain and enter: clarioai.ae
  • Also add: www.clarioai.ae
  • Vercel will show you DNS records to add — keep this screen open

Part B — Update DNS at your registrar

Go to wherever clarioai.ae is registered (likely ae-nic.ae or your Valtrix domain registrar) and add these DNS records:

TypeNameValueTTL
A@76.76.19.613600
CNAMEwwwcname.vercel-dns.com3600
DNS propagation takes time

After saving DNS records, it can take 15 minutes to 48 hours to fully propagate. Vercel shows a green tick in the Domains tab once it's verified. While waiting, use the vercel.app URL to test everything.

Part C — Verify SSL is working

Vercel automatically provisions a free SSL certificate via Let's Encrypt. Once DNS propagates, both https://clarioai.ae and https://www.clarioai.ae will show the green padlock automatically. No action needed from you.

6 Test everything is working

Run through this checklist on the live URL before sharing with anyone:

  • Landing page loads at clarioai.ae — starfield animation runs, nav visible
  • Pricing toggle works — monthly/annual switches correctly
  • "Enter the boardroom" → registration form opens

  • Birthday field detects zodiac sign instantly
  • Sign reveal screen appears after registration
  • 5-question quiz runs correctly
  • Dashboard loads after quiz — greeting shows your name
  • New session — type a decision and submit
  • All 5 advisors respond (this tests the /api/chat proxy)
  • Synthesis generates after all 5 advisors
  • My zodiac cards page — cards display correctly
  • Mobile — open on your phone, check layout
7 How to deploy updates in the future

Every time you push a change to the main branch on GitHub, Vercel automatically deploys it within 30 seconds. No manual steps. The workflow is:

# Make changes to your HTML files, then: git add . git commit -m "Updated pricing copy" git push # Vercel detects the push and deploys automatically # Live in ~30 seconds
Preview deployments

Any branch that is NOT main gets a separate preview URL (e.g. clario-feature-xyz.vercel.app). This means you can test changes on a real URL before pushing to production. Very useful for testing the dashboard or zodiac cards without breaking the live site.
